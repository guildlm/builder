The mechanisms audit — the check that historically found 5 of 7 system bugs by

VERDICT: all benign; NO guarded-shut gate. (A negative result, recorded so the

1. "converged to green on the final deterministic pass"  (src/builder.py:4969)
   REACHABLE RARE SUCCESS PATH. Fires only when the model's LAST fix round leaves
   a purely-mechanical bug that the final deterministic gate fixpoint then repairs
   to green. By the time the final pass runs, the per-round pre-pass has already
   cleared almost every mechanical defect, so this needs the model to introduce a
   fresh mechanical bug in exactly the last round — rare, not dead, not guarded.
   -> KEPT VISIBLE (it is a success log; observing it fire would be informative).

2. "restored go.mod deterministically"  (src/builder.py:4857)
   BENIGN DEFENSIVE GUARD. Reachable in principle (_offending_files matches
   basenames and go.mod is in `written`, and Go emits `go.mod:...` on parse
   errors), but go.mod is generated deterministically from the module path
   (_gomod_content) and is NEVER handed to the model, so it is never malformed ->
   never an offending file -> never restored. Same category as the already-
   excluded "rejected fix of": a guard whose silence is the good outcome.
   -> RECLASSIFIED to _EXPECTED_SILENT in _gate_audit.py.

3. "wrapped the argument to"  (arg-adapter gate, src/builder.py:3757)
   REACHABLE GATE, DEFECT CLASS ABSENT FROM CORPUS — NOT guarded-shut.
   Looked promising: the error shape it keys on ("as X value in argument to Y")

   middleware gate historically. But the 21 hits are the MIDDLEWARE variant
   ("... as Middleware value in argument to Chain", Logging(logger) -> Middleware),

   have logged "wrapped the argument to", which never appears.
   -> KEPT VISIBLE (reachable; re-check if a Store->*API skip ever appears).

NET: 3 unexplained -> 2 (both reachable-but-untriggered, correctly left visible).
