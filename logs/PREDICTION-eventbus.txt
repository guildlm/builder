Written BEFORE the mutation (ordering verifiable vs the log; no wall-clock typed
from memory, per the discipline fix).

Publish sends non-blocking (select + default) so a FULL subscriber (buffer 16)
does not block Publish while it holds the lock — otherwise the whole bus deadlocks.
Close (no test named TestClose) closes all channels.

I PREDICT:
  - Remove `default:` (make the send BLOCKING) -> MISSED (green). No test fills a
    subscriber's 16-buffer without receiving, so the block never triggers. If MISSED,
    it is a real hole: a full/slow subscriber deadlocks Publish under the lock.
  - Make Close a no-op -> likely MISSED (no TestClose checks channels are closed).
IF CAUGHT: a test does pin it and I report the gap as imaginary.

  blocking send (default removed) -> green (MISSED). Real hole: a full subscriber
